# AI / ML Hooks

**21 hook projects.**

## The problem, explained simply

Most dynamic-fee designs (see `dynamic-fees-lp-optimization.md`) are *reactive* — they measure volatility that has already happened and adjust the fee afterward. By the time the fee changes, some of the damage (LVR, a bad trade for LPs) may already be done. The pitch of this category is to be *predictive* instead: use a machine-learning model to forecast near-term volatility or market conditions, and set the fee (or another pool parameter) ahead of time, in principle capturing more value for LPs than a purely reactive formula could.

A second thread in this bucket uses ML/AI not for pricing but for pattern detection — e.g. training a model to recognize the on-chain signature of an MEV bot or a specific trading pattern, then having the hook react accordingly.

## Why fewer teams attack this, and a caution about these claims

Genuinely training, validating, and deploying a useful ML model that outperforms a simple volatility formula is hard, and doing it *on-chain* (where compute is expensive) is harder still — most of these projects run inference off-chain and feed a prediction on-chain via an oracle-like mechanism (sometimes explicitly via an EigenLayer AVS operator running the model, see `staking-restaking.md`). Readers should treat "AI-powered" claims in a hackathon-scale project with healthy skepticism: predicting short-term volatility better than well-known statistical baselines (like GARCH models used in traditional finance) is a genuinely hard, well-studied problem — nobody has reliably solved it, in TradFi or DeFi.

## Common approaches seen in this dataset

1. **Volatility-forecasting dynamic fees** — a model predicts upcoming volatility and sets the fee preemptively rather than reactively (Autopilot Hook, Pooka Valuation Warchest Hook, xtreamly hook).
2. **AI-powered hedging** — combine a volatility/price prediction with an automated hedge (e.g. via perpetual futures) to reduce impermanent loss for LPs.
3. **Sentiment/pattern-based signals** — feed non-price data (social sentiment, on-chain behavior patterns) into a model whose output adjusts pool behavior (SocialPool, Sentiment).

## Prior art / reference material

- GARCH and other classical econometric volatility-forecasting models — the traditional-finance baseline any ML-based volatility prediction should reasonably be compared against.
- EigenLayer AVS model — the common infrastructure pattern for running off-chain ML inference and delivering its output on-chain in a way that's at least partially trust-minimized (see `staking-restaking.md`).
