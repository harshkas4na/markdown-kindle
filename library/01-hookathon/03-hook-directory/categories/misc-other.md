# Misc / Other Hooks

**75 hook projects** — everything left over after the 16 main problem-space categories.

## What's actually in here

Unlike the other category files, this isn't one coherent problem — it's the long tail: hooks whose tags were too generic (just "Custom hooks," "DEX," "DeFi," "Uniswap v4 Hook") to place confidently into a specific problem-space, or that describe pure developer tooling/infrastructure rather than an end-user-facing DeFi product. Recurring sub-clusters within this bucket:

1. **Hook development tooling / meta-infrastructure** — projects that don't solve a trading problem directly but make it *easier to build other hooks* (no-code hook builders, hook composability adapters that let you stack multiple hooks on one pool, hook indexers/explorers).
2. **"Specialized Markets"** — a generic tag used for niche market types that didn't cleanly fit elsewhere (e.g. very specific structured trading venues).
3. **Non-Fungible LP Positions** — treating a liquidity position as a unique, tradeable NFT-like asset rather than a fungible share, an idea distinct from the standard fee/incentive categories.
4. **Gas optimization / gas abstraction** — hooks focused purely on reducing transaction cost or letting users pay gas in a token other than ETH, a cross-cutting concern rather than a trading-strategy idea.
5. **TEE (Trusted Execution Environment) execution** — an alternative to the FHE/ZK privacy approaches seen in `privacy-zk-hooks.md`, using secure hardware enclaves instead of cryptography to hide computation.

## Why this bucket is large

Two forces inflate it: (1) many submissions used only broad, non-descriptive tags, so automatic categorization by tag genuinely can't place them more precisely without reading each description individually, and (2) some entries are one-off, genuinely novel ideas that don't repeat elsewhere in the dataset — which is worth knowing in its own right (see `01-TRACKER.md` for which ideas are common versus rare).

## Prior art / reference material

Because this bucket spans unrelated ideas, there's no single reference body — see the relevant category file (e.g. `privacy-zk-hooks.md` for TEE/FHE-adjacent entries, `liquidity-management-incentives.md` for NFT-position entries) for closer prior art where applicable.
